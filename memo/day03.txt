1. 자료형(Type) : 값의 종류

	자료형	type	byte	값

	 정수형	int	4	양의 정수, 음의 정수, 0
		long	8	양의 정수, 음의 정수, 0L(etc..1L,10L,100L)

	 실수형	double	8	소수점이 있는 수
		float	4	소수점이 있는 수(etc..3.14f, 5.2f. -15.4f)
	
	 논리형	boolean	1	true / false

	 문자형	char	2	문자 1개('a','b','c'...)

	 문자열	String	?	"a", "abc", "true", "12"...

2. 변수(variable)

	어떤 값을 저장하기 위한 저장공간
	
	1) 변수의 특징
		-변수는 오직 하나의 자료형만 저장할 수 있다.
		-변수는 오직 하나의 값만 저장할 수 있다.

	=> 하나의 자료형과 하나의 값만 저장할 수 있는 저장공간이다.

	2) 변수의 선언과 사용
		변수의 선언
			자료형 변수명 ;
			int number;
			boolean isTrue;
			char alpha;
			
		변수에 값을 저장하는 방법
			변수명 = 값;
			= 대입연산자(오른쪽의 값을 왼쪽에 저장한다.)
			number = 10;(10을 number 변수에 저장)
		
		변수의 선언과 동시에 초기화
			자료형 변수명 = 값;
			int number = 100;
			char alpha = 'a';

		여러개의 변수를 선언하는 방법
			int num = 10;
			int num2 = 20;
			int num3 = 30;
			int num1 , num2=20 , num3=30 ;

		
 	+) 저장공간과 값
		int num = 10; // 저장공간
		num = 10 + num 	// 10 + 10 (왼쪽 num은 저장공간, 오른쪽 num은 값)
		System.out.println(num)  // 20;


	3) 변수 선언시 주의사항
		같은 영역(중괄호 안) 같은 이름의 변수를 선언할 수 없다.
		되도록 소스코드 위쪽에 한꺼번에 선언한다.
		되도록 선언과 초기화를 동시에 하는 것이 좋다.


	4) 변수명 주의사항
		문자부터 시작한다.
		특수문자를 사용할 수 없다(단, _ , $는 가능)
		공백을 사용할 수 없다(카멜 표기법,스네이크 표기법 등을 활용)
		키워드(예약어)를 변수명으로 사용할 수 없다.
		되도록이면 소문자(대소문자로 구분)로 시작하는것이 좋다.
		되도록 의미 있는 단어,문자를 사용한다.
		되도록 한글을 사용하지 않는다.

3. 상수(constant)
	한번 값이 지정되면 변경할 수 없는 데이터를 의미
	final 키워드를 사용하여 상수를 정의할 수 있다.
	
	1) 상수 선언
		final 자료형 상수명 = 값;
		상수명은 대문자로 작성하고 단어가 바뀔때는 _로 연결한다.
	2) 상수 사용시 주의사항
		- 이름 작성 규칙
		- 초기화 필수 : final 키워드 사용시 선언과 동시에 초기화 해야한다.
		- 불변성 : 한 번 값이 지정되면 변경할 수 없으므로 값을 유지할 때 쓴다.

4. 변수를 사용하는 이유 (값을 왜 변수에 저장해서 사용하는가?)
	반복되는 값을 쉽게 관리하기 위해서(값을 재사용하기 위해서)
	값에 의미를 부여하기 위해(값에 이름을 붙여준다.)
	=> 프로그램의 가독성, 유지보수성, 재사용성을 위해 변수를 사용합니다.

6. 제어문자 (Escape Character / Sequence)
	따옴표 안에서 사용되며 미리 예약된 특수한 기능을 하는 문자
	\(백슬래쉬, 역슬래쉬) 를 사용한다.
	\n : 줄바꿈 
	\t : 탭키와 같은 역할
	\" : 문자열 안에서 큰따옴표를 표현
	\' : 문자열 안에서 작은따옴표를 표현
	\\ : 문자열 안에서 \를 표현
	
7. 출력메소드 
	소괄호안에 있는 내용을 콘솔창에 출력한다.
	println()
		전달된 값을 콘솔창에 출력하고 줄바꿈이 된다.
	print()
		전달된 값을 콘솔창에 출력하고 줄바꿈이 되지 않는다.
	printf()
		형식(format)에 맞는 값을 출력하고 줄바꿈이 되지 않는다.
	

	1) 출력 메소드의 목적
		개발자가 오류를 해결하기 위해 값을 확인할 때 사용한다.
	-디버깅 : 개발자가 프로그램 중간에 변수값등을 출력하여 오류를 확인하거나, 데이터의 흐름을 추적
	-정보전달 : 프로그램의 진행상태나 결과를 알림
	-형식 지정 출력 : 데이터를 정렬하거나 특정 형식으로 표현
	
	2) printf()
		형식 지정자를 사용하여 데이터를 특정 형식으로 출력할 수 있다.
		줄바꿈은 자동으로 이루어지지 않으며, 필요하면 \n을 추가해서 사용한다.
		
	3) 형식 지정자
		출력 형식을 지정한다.
		%로 시작하며 printf()에서만 사용한다.
		반드시 따옴표 안에서 작성한다.
		, 뒤에 나오는 값이 형식 지정자의 위치로 대입된다(들어가게 된다)
		
		자료형	형식지정장		의미

		int	%d		decimal(10진수 정수)
			%o		octal(8진수 정수)
			%x		haxadecimal(16진수)

		double	%f		float
		
		char	%c		character(문자)
		String	%s		string(문자열)

	4) 서식문자의 옵션
		%[- or 0][n][.m]서식문자
		-	왼쪽 정렬
		0	비어있는 공간을 0으로 채움(int,double만 가능)
		n	자리수
		.m	소수점 아래 자리수 / 문자열 자르기

8. 형변환 (type casting)
	하나의 자료형을 다른 자료형으로 변환하는 과정을 의미한다.
	강제 형변환과 자동 형변환 두가지 방식이 있다.

	1)강제 형변환
	(자료형) 깂
	큰 자료형을 작은 자료형으로 변환할 때 사용된다.
	데이터 손실이 발생할 수 있다.
	ex) double -> int, long -> int
	
		- 실수 -> 정수		소수점 이하의 값이 버려짐(반올림x)
		- 큰 정수 -> 작은 정수 	데이터 손실 또는 값의 왜곡이 발생
		- 문자 -> 정수 		문자를 정수(유니코드/ascii코드) 로 변환
		- 정수 -> 문자		정수를 문자코드 값으로 변환

	2)자동 형변환
	작은 크기의 자료형을 큰 크기의 자료형으로 변환할 때 컴파일러가 자동으로 수행한다.
	데이터 손실이 없기 때문에 안전하게 변환할 수 있다.
	명시적으로 형변환 연산자를 사용하지 않아도 된다.
		-작은 자료형에서 큰 자료형으로 변환
		-연산 시 형변환이 이루어질 때 자동변환(더 큰 자료형으로 자동변환)
		-문자형(char)에서 정수형(int)로 변환
		-정수형값을 실수로 변환

	3)문자열의 형변환
	- 다른 자료형을 문자열로 변환
		문자열과 다른 자료형을 연결(+)하면 결과는 항상 문자열이 된다.
	- 문자열을 다른 자료형을 변환
		기본 자료형은 기본 자료형 끼리만 형변환 가능
		String 클래스 타입(레퍼런스 타입)

		모든 기본 자료형을 담고 있는 클래스 타입이 있고 String 다른 자료형으로 변환시는 이 클래스 타입의 도움을 받는다.	
		int -> Integer
		long -> Long
		double -> Double
		float -> Float
		char -> Character
		bool -> Boolean
		
		String data1 = "10";
		String data2 = "abc";
		int result = Integer.parseInt(data1);
		long result = Long.parseLong(data1);
		double result = Double.parseDouble(data1);
		char result4 = "abc".charAt(0)
